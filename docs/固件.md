# 固件

### 🔧 **Bootloader & Pre-OS 阶段**

#### 1. **uboot.img**
- **作用**：U-Boot（Universal Bootloader）镜像，是设备上电后首先运行的引导程序。
- **内容**：包含初始化 DDR、eMMC、USB、显示等硬件的代码，负责加载下一阶段（如 `boot.img` 或 `resource.img`）。
- **平台相关**：Rockchip 平台通常使用 U-Boot 的定制版本。

#### 2. **resource.img**
- **作用**：Rockchip 特有镜像，包含设备树（Device Tree Blob, `.dtb`）、启动 logo、内核参数等资源。
- **内容**：
    - 多个 `.dtb` 文件（适配不同硬件变体）
    - 启动画面（splash screen）
    - 内核 cmdline 参数
- **加载时机**：由 U-Boot 加载并解析，用于配置内核启动环境。

#### 3. **baseparameter.img**
- **作用**：Rockchip 平台特有的参数分区，存储设备基础配置信息。
- **内容**：可能包括串号、MAC 地址、硬件版本、DDR 配置等只读或可写参数。

#### 4. **misc.img**
- **作用**：用于存储 bootloader 与 recovery/OTA 之间的通信标志。
- **典型用途**：标记是否进入 recovery、OTA 更新状态等。

---

### 🐧 **Linux Kernel & Early Init**

#### 5. **boot.img**
- **作用**：Android 主启动镜像，包含 Linux 内核 + 初始 ramdisk（initramfs）。
- **内容**：
    - `kernel`：Linux 内核（zImage/Image.gz）
    - `ramdisk`：初始根文件系统（包含 `/init`、基本驱动、ueventd 等）
    - `dtb`（有时内嵌）
- **启动流程**：U-Boot → `boot.img` → 内核启动 → 挂载 `ramdisk` → 执行 `/init`

#### 6. **init_boot.img**（Android 12+ 引入）
- **作用**：将传统 `boot.img` 中的 `ramdisk` 拆分为独立镜像，用于支持 Generic Kernel Image (GKI)。
- **内容**：仅包含 initramfs（不再包含内核）。
- **配合使用**：与 `vendor_boot.img` 协同工作。

#### 7. **vendor_boot.img**（Android 12+ GKI 架构）
- **作用**：包含 vendor-specific 的内核模块、设备树、bootconfig 等。
- **内容**：
    - `dtb` / `dtbo`
    - `vendor_ramdisk`（含 vendor init 脚本、ko 模块）
    - `bootconfig`
- **目的**：实现内核与 vendor 实现解耦，便于 Google 统一维护通用内核。

#### 8. **dtbo.img**（Device Tree Overlay）
- **作用**：动态设备树叠加层，用于在不修改主 dtb 的情况下调整硬件配置。
- **内容**：`.dtbo` 文件，描述特定外设（如摄像头、传感器）的连接方式。
- **加载时机**：由 bootloader 或 kernel 在启动时应用。

---

### 📦 **Android System 分区镜像**

#### 9. **system.img**
- **作用**：Android 核心系统分区，包含 AOSP 框架、系统应用、库等。
- **内容**：
    - `/system/bin/`, `/system/lib/`, `/system/app/`
    - Android Framework（ActivityManager, PackageManager 等）
    - 系统 APK（Settings, Launcher, Dialer 等）

#### 10. **system_ext.img**
- **作用**：扩展 system 分区，用于存放 OEM/ODM 添加的系统级组件（避免修改原生 system）。
- **内容**：厂商定制的系统服务、权限配置、系统 APK 扩展等。

#### 11. **product.img**
- **作用**：产品特定配置和应用，与具体设备型号绑定。
- **内容**：
    - 产品专属 APK（如 MAXHUB 会议软件）
    - 产品配置（`/product/etc/permissions/`）
    - 品牌资源（壁纸、铃声等）

#### 12. **vendor.img**
- **作用**：SoC 厂商（如 Rockchip）或 ODM 提供的闭源驱动和 HAL 实现。
- **内容**：
    - GPU 驱动（Mali）
    - Camera HAL、Audio HAL
    - Wi-Fi/BT 固件
    - `/vendor/lib/hw/` 下的 .so 文件

#### 13. **odm.img**
- **作用**：ODM（原始设计制造商）定制分区，比 vendor 更贴近具体设备。
- **内容**：设备特定的 HAL、配置文件、固件（如特定摄像头模组驱动）。

---

### 🔌 **动态内核模块（DLKM）分区（Android 12+）**

> DLKM = Dynamically Loadable Kernel Modules

#### 14. **system_dlkm.img**
- **作用**：存放可动态加载的内核模块（.ko），属于 system 命名空间。
- **用途**：减少 boot.img 大小，按需加载模块。

#### 15. **vendor_dlkm.img**
- **作用**：vendor 提供的动态内核模块。

#### 16. **odm_dlkm.img**
- **作用**：ODM 提供的动态内核模块。

> 这些 DLKM 镜像在启动后由 `init` 挂载到 `/system/lib/modules/` 等路径，并通过 `modprobe` 加载。

---

### 🔒 **安全与验证**

#### 17. **vbmeta.img**
- **作用**：Android Verified Boot（AVB）元数据镜像。
- **内容**：包含对 `system`、`vendor`、`boot` 等分区的哈希值和公钥签名。
- **目的**：确保系统完整性，防止未授权修改（若开启 AVB）。

---

### 🖼️ **其他专用镜像**

#### 18. **logo.img** / **bootvideo.img**
- **作用**：开机 Logo 或动画视频。
- **加载时机**：由 U-Boot 或 early init 显示。

#### 19. **firmware.img**
- **作用**：包含各类硬件固件（非 Android 层），如：
    - DSP 固件
    - NPU 微码
    - 电源管理芯片固件
- **注意**：此文件可能是 Rockchip 打包的完整 firmware bundle。

#### 20. **super.img**
- **作用**：动态分区（Dynamic Partitions）的超级镜像容器（Android 10+）。
- **内容**：包含 `system`、`vendor`、`product` 等逻辑分区的合并镜像。
- **刷机方式**：使用 `fastboot update super.img` 或 `lpmake` 工具拆分刷入。
- **你的情况**：你同时有独立分区镜像（如 system.img）和 super.img，说明该 OTA 包支持两种刷机方式。

### 总结表格

| 镜像 | 类型 | 作用 |
|------|------|------|
| `uboot.img` | Bootloader | 硬件初始化，加载 OS |
| `resource.img` | Rockchip 资源 | dtb、logo、cmdline |
| `boot.img` | Kernel + Init | 主启动镜像（旧架构） |
| `init_boot.img` | Initramfs | GKI 架构下的 init |
| `vendor_boot.img` | Vendor Kernel | GKI 下的 vendor 内容 |
| `dtbo.img` | 设备树叠加 | 动态硬件配置 |
| `system.img` | AOSP 系统 | 核心 Android 框架 |
| `system_ext.img` | 系统扩展 | OEM 系统组件 |
| `product.img` | 产品定制 | MAXHUB 专属应用 |
| `vendor.img` | SoC 厂商 | Rockchip 闭源 HAL |
| `odm.img` | ODM 定制 | 设备特定驱动 |
| `*_dlkm.img` | 动态内核模块 | 可加载 .ko 模块 |
| `vbmeta.img` | 安全验证 | AVB 签名与哈希 |
| `super.img` | 动态分区容器 | 合并多个逻辑分区 |
